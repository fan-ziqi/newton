class newton.sim.ik.IKSolver(model, joint_q, objectives, *a, **kw)[source]#

Modular inverse-kinematics solver.

The solver uses an adaptive Levenberg-Marquardt loop and supports three Jacobian back-ends:

  • AUTODIFF ‚Äî Warp‚Äôs reverse-mode autodiff for every objective

  • ANALYTIC ‚Äî objective-specific analytic Jacobians only

  • MIXED ‚Äî analytic where available, autodiff fallback elsewhere

Parameters:

  • model (newton.Model) ‚Äì Singleton articulation shared by _all_ problems.

  • joint_q (What varies from problem to problem are (1) the corresponding row in) ‚Äì Initial joint coordinates, one row per problem. Modified in place.

  • objectives ((2) any per-problem data stored internally by the) ‚Äì Ordered list of objectives shared by _all_ problems. Each IKObjective instance can carry arrays of per-problem parameters (e.g. an array of target positions of length n_problems).

  • jacobian_mode (JacobianMode, default JacobianMode.AUTODIFF) ‚Äì Backend used in compute_jacobian.

  • lambda_initial (float, default 0.1) ‚Äì Initial LM damping per problem.

  • lambda_factor (float, default 2.0) ‚Äì Multiplicative update factor for Œª.

  • lambda_min (float, default 1e-5) ‚Äì Lower clamp for Œª.

  • lambda_max (float, default 1e10) ‚Äì Upper clamp for Œª.

  • rho_min (float, default 1e-3) ‚Äì Acceptance threshold on predicted vs. actual reduction.

Supported joint types#

ANALYTIC and MIXED modes currently only support models with revolute, prismatic, and fixed joints. For more complex joint types (e.g., free, ball, D6), use AUTODIFF mode. Velocity-space optimization is planned for broader analytic support in future updates.
